Git & GitHub

Git is a free and open-source distributed version control system designed to handle projects of any size with speed and efficiency.

What You Can Do in Git

  • Frictionless Context Switching: Create branches, commit changes, switch between branches, and merge seamlessly.
  • Role-Based Codelines: Maintain branches for production, testing, and daily work.
  • Feature-Based Workflow: Create branches per feature, work independently, and merge when ready.
  • Disposable Experimentation: Try experiments in isolated branches without affecting main code.

Common Git Commands

Command Description
git clone Clone a repository to your local machine
git init Initialize a new Git repository
git add Stage changes for commit
git commit Save changes to the repository
git push Upload commits to a remote repository
git pull Fetch and merge changes from a remote repository

GitHub

GitHub is a developer platform that uses Git for version control. It allows developers to store, manage, and share code, while also providing access control, issue tracking, project management, and wikis.

What's Inside GitHub?

Feature Description
Projects Organize tasks and track progress
Repository Store your code and version history
Pages Host websites from your repository
Issues Track bugs and feature requests
Pull Requests Propose and discuss code changes before merging
Codespaces Develop in a cloud-hosted environment
